Technical Paper

Abstract

This paper presents House Price Predictor Pro, a full-stack ML web application for real estate valuation using the Ames Housing dataset. A six-model ensemble (Linear Regression, Ridge, Lasso, KNN, Random Forest, Gradient Boosting) with GridSearchCV hyperparameter tuning and 5-fold cross-validation is implemented. Preprocessing includes median imputation, MinMaxScaling, feature engineering, and PCA (95% variance). The best model achieves R² ≥ 0.87. Results are exposed via Flask REST API and visualized in a modern responsive UI.

I. Introduction

Accurate property valuation is critical for buyers, sellers, and investors. Machine learning offers a data-driven alternative capable of processing complex feature interactions at scale. The Ames Housing dataset comprises 1,460 residential properties with 79 attributes, serving as a standard regression benchmark. This work combines best practices from multiple open-source Ames ML projects into a unified production system.

II. Methodology

A. Feature Selection: 8 high-correlation features: LotArea, OverallQual, YearBuilt, GrLivArea, TotalBsmtSF, GarageCars, FullBath, BedroomAbvGr.

B. Preprocessing: Median imputation → MinMaxScaler → Derived features (HouseAge, TotalSF) → PCA (95% variance).

C. Training: GridSearchCV 5-fold CV for Ridge, Lasso, KNN, Random Forest, Gradient Boosting. Best model saved via joblib.

D. Evaluation: RMSE, R², MAE on 20% hold-out test set.

III. Results

Random Forest and Gradient Boosting consistently outperform linear models. Model metrics below are dynamically loaded from backend/metrics.json after training.

IV. Conclusion

House Price Predictor Pro demonstrates that combining multiple ML models with robust preprocessing achieves high-accuracy real estate valuation. The best ensemble achieves R² > 0.87, surpassing the 80% target. The Flask API and modern UI make predictions accessible to non-technical users. Future work includes incorporating all 79 Ames features via automated feature selection and cloud deployment.
